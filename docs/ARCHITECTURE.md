# JISO アーキテクチャ

## 入力
人間が入れるのは「やりたいこと」一度だけ。以降は稟議の承認/却下のみ。

## 4層
1. **Goal Compiler** — やりたいこと → KPIツリー（funnel係数で逆算）
2. **実行エージェント群** — 営業(架電/商談) / マーケ(広告/SNS/生成) / バックオフィス(請求/督促)
3. **稟議エンジン** — 費用発生アクションは pending で停止。承認後に執行、実測と突き合わせ
4. **Human Task Queue** — オフライン実行・物理購買はAIが人間に手順書+費用つきで発注

## セキュリティ
- エージェントごと最小権限 / 外部通信は許可リスト制
- 外部スキル/MCP導入は SkillSpector スキャン + ポリシー判定を通過したもののみ
- 全アクション audit_log 記録 / 決済情報はエンジン本体から分離
