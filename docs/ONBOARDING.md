- [ ] 適格請求書発行事業者の登録番号（インボイス制度。請求書発行に必須）
- [ ] 決算期・消費税の課税/免税区分（価格表示と請求書の税計算に使用）
- [ ] **受取用**の銀行口座情報（銀行名/支店/種別/番号/名義カナ）
      ※これは請求書に書く「振込先」なので渡してOK。ログイン情報とは別物
- [ ] ブランド名の最終決定（国内/海外）とドメイン
- [ ] 特商法ページ・プライバシーポリシーの有無（なければOSが生成）

## Tier 2｜金流（ここだけ設計が命）
### 受け取る（売上）— OSがほぼ全自動化できる
- [ ] Stripeアカウント（あなたが開設→**制限付きAPIキー**を共有）
      → OSが請求書・決済リンクを自動発行、入金をWebhookで自動検知
- [ ] ココナラ/Freelancerは内蔵決済 → 出金先口座の登録だけあなた
- [ ] 請求書「発行」: 完全自動化可（Tier 0の情報＋Stripe or 自前PDF生成）

### 支払う（経費）— 「準備は自動・最後のボタンは人間」が現実解
- [ ] 法人カード: あなたが各広告アカウント(Google/Meta)に登録（カード番号はOSに渡さない）
      → 以降OSはAPIで「承認されたAR金額」を予算上限として書き込むだけ
- [ ] 請求書「払い」: できる。フローは——
      OSが請求書PDFを受領・読取 → 承認済みARと金額照合 → 照合OKなら
      振込情報を整形してSlackへ「この内容で振り込め」と人間タスク化 →
      あなたが銀行アプリで実行（1分）→ OSが消込・記帳
      ※照合NG（ARなし/金額超過）は支払い保留として警告
- [ ] （任意・後日）GMOあおぞらネット銀行などの法人API口座:
      残高・入出金の自動照会から接続。振込「実行」のAPI接続は
      実績が貯まってから閾値付きで——初日からは推奨しない

### 渡してはいけないもの（OSにもチャットにも絶対に貼らない）
- 銀行のログインID/パスワード/ワンタイムパスワード
- カード番号・CVV
- マイナンバー、印鑑、印鑑証明、本人確認書類
- 各サービスの「マスターパスワード」（渡すのは権限を絞ったAPIキーのみ）

## Tier 3｜事業別（物販実験の場合）
- [ ] 販売アカウント: メルカリShops / Amazonセラー / eBay（開設は本人確認が要るためあなた）
- [ ] 仕入れアカウント: アリババ/1688等（同上）
- [ ] 発送手段の選択: 自己発送（あなたが梱包）or FBA/メルカリ便
- [ ] 仕入れ予算上限: ¥100,000（DB登録済み・フェーズゲート付き）

## キーの取り扱いルール
1. すべて最小権限・短い有効期限で発行（例: GitHubはリポジトリ1個のContentsのみ）
2. チャットに貼ったキーは作業完了後に失効・再発行
3. OSが使った全キーの用途は audit_log に記録される
